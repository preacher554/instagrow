# InstaGrow Research Engine v2

## Purpose

This playbook clarifies how the InstaGrow Research Layer should think, validate, synthesize, and hand off work. It does not replace `playbooks/instagram-research-layer.md` or `playbooks/cross-platform-research.md`; it strengthens them with a stricter research engine so sub-agents do not produce vague reports.

Research is complete only when it produces:

1. evidence-backed insight,
2. confidence level,
3. psychological mechanism,
4. pattern/gap/tension synthesis,
5. monetization relevance,
6. Instagram-native content briefs,
7. experiment cards and learning questions.

## Canonical Research Sequence

Every serious research run follows this exact order:

```text
0. Account Status Routing
1. Brand Intelligence Card
2. Research Brief
3. Cross-Platform Signal Collection
4. Source Normalization
5. Epistemic Validation
6. Psychological Interpretation
7. 10+3 Content Deconstruction for key examples
8. Pattern Recognition
9. Gap Analysis
10. Tension Map
11. Temporal / Lifecycle Assessment
12. Opportunity + Monetization Scoring
13. Research Report
14. Content + Experiment Handoff
15. Data Store Updates
```

Do not jump directly from source collection to recommendations unless Chief explicitly asks for a quick scan. Even in quick mode, label uncertainty and missing evidence.

## Operating Depth

### Quick Scan

Use when Chief needs direction quickly.

Minimum:
- account status,
- brand/context assumptions,
- Instagram + one external platform,
- 5+ source notes,
- top 3 opportunities,
- confidence tags on major claims,
- 1–3 recommended next experiments.

### Normal Run

Default for client research.

Minimum:
- completed Brand Intelligence Card,
- Instagram + two external platforms,
- 15+ source notes,
- source normalization,
- opportunity scoring,
- bias disclosure,
- 3 Reels, 2 carousels, 5 Stories,
- experiment backlog.

### Deep Run

Use for major strategic decisions.

Minimum:
- Instagram + X/Twitter + TikTok + YouTube + web/forum when accessible,
- 30+ source notes,
- 10+3 analysis on at least 5 strong examples,
- pattern/gap/tension map,
- temporal lifecycle assessment,
- 90-day research implications,
- data store updates.

## Brand Intelligence Requirement

Before research, fill `templates/brand-intelligence-card.md` or state explicit assumptions.

Required fields:
- brand/account,
- account status,
- niche/sub-niche,
- offer or monetization path,
- target audience,
- business goal,
- growth goal,
- brand voice and anti-voice,
- direct competitors,
- aspirational competitors,
- constraints.

If these are missing and materially affect strategy, ask Chief. If they are missing but can be safely assumed, proceed with labelled assumptions.

## Epistemic Validation Engine

Every major claim, insight, risk, and recommendation must be tagged:

- `[VERIFIED]` — directly supported by observed source data from this run.
- `[PROBABLE]` — pattern is visible but sample size or variables are limited.
- `[HYPOTHESIS]` — strategically logical but not proven for this client yet.
- `[SPECULATION]` — useful for brainstorming or monitoring, not for major decisions.

Do not tag every sentence. Tag major bullets and decisions so the report stays readable.

### Required Bias Disclosure

Every normal/deep report must include `Data Limitations & Bias`:

- survivorship bias: only winners/viral posts were sampled,
- recency bias: sources are concentrated in a short recent window,
- selection bias: sources overrepresent one format, platform, creator type, or region,
- confirmation bias: evidence was selected to support a pre-existing belief,
- access bias: private metrics such as saves, shares, retention, and Instagram Insights were unavailable.

Explain how the bias changes confidence.

## Anti-Generic Quality Gate

Reject or rewrite generic claims. These phrases are not allowed unless explained with evidence and mechanism:

1. “engagement tinggi” / “performa bagus”
   - specify compared to what, which metric, and why it matters.
2. “kontennya menarik”
   - specify hook type, psychological mechanism, and retention driver.
3. “audiens aktif”
   - specify visible behavior: comments, saves, shares, questions, DMs, profile visits, or repeated response pattern.
4. “kompetitor kuat”
   - specify strong in what: positioning, views, authority, consistency, conversion, community, or format mastery.
5. “ikuti trend”
   - specify lifecycle, saturation, brand fit, and entry angle.
6. “perbanyak edukasi”
   - specify topic, format, audience problem, cadence, and success metric.
7. “lebih konsisten”
   - specify cadence, format mix, posting window, and what consistency is intended to teach the algorithm/audience.

Replacement rule:

```text
Generic claim → Evidence → Mechanism → Implication → Action
```

## Psychological Interpretation Engine

For key examples and recommendations, explain why the audience would respond.

Use these engagement motivations:

- comments: disagreement, confession, shared experience, identity threat, question, humor, community belonging;
- shares: social currency, altruism, identity expression, “this is me/us,” utility for a friend, controversy;
- saves: future reference, checklist utility, framework, template, aspiration, “I will need this later”;
- watch completion: curiosity gap, reveal, tension, story payoff, fast utility, aesthetic reward, fear of missing the point;
- follows: consistent value, niche clarity, authority, identity alignment, parasocial trust, aspirational proximity.

Do not say “the hook is strong” without naming the mechanism.

## Pattern Recognition

Classify findings into four pattern types:

- recurring pattern: same signal appears repeatedly;
- correlation pattern: variables appear together, but causation is not assumed;
- anomaly pattern: source over/under-performs against expectation;
- absence pattern: something important is missing from competitors or the market.

Pattern claims require:
- source count or examples,
- confidence tag,
- strategic implication,
- falsifiability note: “this pattern weakens if…”

## Gap Analysis

Use three gap types:

- content gap: unanswered topic/angle,
- positioning gap: unclaimed audience perception,
- execution gap: underused format/style/distribution approach.

Each gap must include:
- evidence,
- why it exists,
- impact 1–10,
- feasibility 1–10,
- urgency,
- decay risk,
- first experiment.

## Tension Map

Actively look for strategic contradictions:

- data vs strategy,
- audience claims vs audience behavior,
- short-term growth vs long-term brand,
- differentiation vs familiar format expectation,
- competitor success that contradicts best practice.

Resolve each tension by one of:
- integration: both are true in different contexts,
- hierarchy: one priority dominates now,
- third option: a better path combines or bypasses both,
- accept ambiguity: evidence is insufficient; define a test.

## Temporal Intelligence

Every trend, format, or major recommendation should receive:

### Lifecycle

- pioneering,
- emerging,
- growing,
- saturated,
- declining,
- dead.

### Urgency

- `[ACT NOW]` — window likely under 30 days,
- `[ACT SOON]` — 30–90 days,
- `[BUILD TOWARD]` — strategic for 3–12 months,
- `[MONITOR]` — watch, not yet execute.

### Strategy Type

- spike: fast reach, weak compounding,
- compounding: slower, builds durable asset,
- hybrid: spike that can become compounding with follow-up.

## Opportunity + Monetization Scoring

Use `templates/opportunity-scorecard.md`.

Score 0–5:
- pain intensity,
- business relevance,
- engagement proof,
- novelty,
- Instagram format fit,
- client fit,
- trust potential,
- lead potential,
- execution ease,
- saturation penalty.

Formula:

```text
score = pain_intensity
      + business_relevance
      + engagement_proof
      + novelty
      + instagram_format_fit
      + client_fit
      + trust_potential
      + lead_potential
      + execution_ease
      - saturation_penalty
```

Decision bands:

- 32+ immediate priority,
- 24–31 strong test candidate,
- 16–23 backlog / sharpen angle,
- below 16 archive unless strategic context changes.

## Recommendation Completeness

Every major recommendation must include:

- WHAT: exact action,
- WHY: linked insight/evidence,
- HOW: execution steps,
- WHO: owner role,
- WHEN: timeline/frequency,
- MEASURE: success metric and timeframe,
- CONSTRAINT: what not to do,
- DEPENDENCIES: what must exist first,
- RISK: what could go wrong,
- NEXT STEP: immediate action.

## Required Final Handoff

Use `templates/research-report.md` at the end of a run.

The report must include:
- executive summary,
- scope and source coverage,
- confidence summary,
- data limitations and bias,
- top opportunities with scorecards,
- psychological mechanisms,
- pattern/gap/tension findings,
- temporal/lifecycle notes,
- recommended Reels, carousels, Stories,
- experiment cards,
- data store updates,
- next action for the Content System Layer.

## Sub-Agent Contract

When Yuya spawns a research sub-agent, the sub-agent must receive:

1. client/brand context or explicit assumptions,
2. account status or instruction to classify first,
3. research objective and depth: quick, normal, or deep,
4. platforms to inspect,
5. source minimum,
6. required output template,
7. instruction to use confidence tags and bias disclosure,
8. instruction to update data stores only if asked or if operating inside the repo.

A sub-agent should never return only “ideas.” It must return evidence, reasoning, scores, and handoff-ready next actions.
